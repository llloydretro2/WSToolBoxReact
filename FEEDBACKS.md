# FEEDBACK.md

1. 顶部的手卡窗口应该增加一个一键清空的按钮，方便玩家快速清空手卡。

   **已实现（2026-05-16）：** 在 `FixedHandBar` 右侧添加了垃圾桶图标按钮（`DeleteOutlineIcon`）。点击后调用 `handleClearAll`，同时清空暗手、副露、副露构建区和分析结果。

2. 对于某一个牌形，如果用户的输入已经达成了，不应该显示高可行和路线，而是直接显示已达成，并不需要提供更多路线意见

   **已实现（2026-05-16）：** 当分析结果 `hand.isComplete === true`（向听数为 -1）时，役种路线分析面板整体替换为 `CompletedHandPanel`，显示"已和牌"提示及所有已达成役种，不再展示路线建议卡片。

3. 移除手气排面选择器按钮，这个设计经过实际使用之后发现没有什么实际意义

   **已实现（2026-05-16）：** 从 `MahjongTilePicker` 中完全移除了"暗手 / 副露"模式切换 `ToggleButtonGroup`。主牌格始终将点击添加到暗手（左键加、右键减）；副露构建区改为常驻展示，内置独立的缩小版牌格，点击直接向副露暂存区添加牌，无需切换模式。相关的 `pickerMode` 状态和 `onModeChange` prop 一并删除。

4. 顶部窗口的设计可以修改一下，现在多个手牌状态是横住放的而不是竖着，比如说"门清"和"听牌"是横着并列的，这样并不节省空间也不直观，改成竖着放这些状态，并且如果是完整和牌状态，也增加和的包括的役作为一个状态显示在顶部手卡窗口

   **已实现（2026-05-16 / 2026-05-17）：** `FixedHandBar` 的状态标签区由 `direction="row"` 改为 `direction="column"`，"门清/副露"与"听牌/X向听"等标签现垂直堆叠。役种显示逻辑扩展为：只要手牌中已满足某役种结构（不限于完整和牌），即在状态列显示对应役名标签。具体判定标准：引擎标记为 `CONFIRMED` 的役，或标记为 `HIGH` 且 `en.needed` 为空或以 "Keep" 开头的役（说明该役结构已具备，仅需等待整体完牌），统一归入 `achievedRoutes` 在顶部显示。

   **追加（2026-05-17）：** 在役种路线分析卡片中新增"已达成（Achieved）"可行度等级，位于"已确立"与"高可行"之间，以深绿色（`#52b788`）区分。凡满足上述达成条件的役种，分析卡片的可行度标签从"高可行"升级为"已达成"，`HandStatusPanel` 中的确立役种区块也同步改用 `achievedRoutes` 展示，标题更新为"已达成役种"。
